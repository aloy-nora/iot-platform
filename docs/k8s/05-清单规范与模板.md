# 05 · 清单规范与模板

## 文件命名规范

- 放在 `deploy/k8s/`，**数字前缀控制 `apply` 顺序**：
  - `00-namespace.yaml` → `10-mysql.yaml` → `20-redis.yaml` → `30-...`
- 先建依赖（namespace），再建依赖它的资源，避免 `not found`（见 `03-问题排查` 第 1 条）。

## 字段顺序规范（所有清单统一）

| 层级 | 固定顺序 |
|---|---|
| 文件内对象 | Secret → PVC → Deployment → Service |
| 对象头 | `apiVersion` → `kind` → `metadata` → `spec`（Secret 为 `type` → `stringData`） |
| metadata | `name` → `namespace` |
| container | `name` → `image` → `command`/`args` → `env` → `ports` → `volumeMounts` → `readinessProbe` → `livenessProbe` |

> `command` 与 `args` 占同一槽位（`image` 之后、`env` 之前）。
> 用 `args`：只给镜像默认入口加参数（如 MySQL 的字符集，密码走原生 env）。
> 用 `command: ["sh","-c",...]`：需要 shell 展开变量来注入密码（如 Redis）。

## 密码注入的两种正确姿势

**A. 镜像原生支持 env（如 MySQL）** —— 直接 env 引 Secret，不用 shell：
```yaml
env:
  - name: MYSQL_ROOT_PASSWORD
    valueFrom:
      secretKeyRef: { name: mysql-secret, key: MYSQL_ROOT_PASSWORD }
```

**B. 密码只能走命令行参数（如 Redis）** —— 用 `sh -c` 让 `$VAR` 展开：
```yaml
command: ["sh", "-c", "exec redis-server --requirepass \"$REDISCLI_AUTH\" --appendonly yes"]
env:
  - name: REDISCLI_AUTH        # 一变量两用:server 的 requirepass + redis-cli 自动认证(探针免写明文)
    valueFrom:
      secretKeyRef: { name: redis-secret, key: REDIS_PASSWORD }
```
> `REDISCLI_AUTH` 本是 redis-cli 的客户端变量，这里借它同时给 server 当密码，省掉一个重复变量。
> 若更看重语义清晰，也可保留 `REDIS_PASSWORD` 给 server、探针改用 `["sh","-c","redis-cli -a \"$REDIS_PASSWORD\" ping"]`。二选一即可。
- 关键点：`args:[...]` 是纯字符串数组，**不过 shell，`$VAR` 不展开**；要展开必须 `sh -c`。
- `exec` 前缀：让主进程顶替 shell 成 PID 1，能收到 `SIGTERM` → 优雅停机。
- 目标：清单里搜不到明文密码，全靠 Secret。

## 非 root 运行（生产化时加）

```yaml
spec:
  template:
    spec:
      securityContext:
        runAsUser: 999      # 容器内业务用户的 uid
        fsGroup: 999        # 挂载卷属组=此值，降权后才写得进 PVC
```

## 通用模板（Secret + PVC + Deployment + Service）

把 `NAME`、`IMAGE`、`PORT`、`MOUNT` 换成实际值即可。探针按服务改。

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: NAME-secret
  namespace: iot
type: Opaque
stringData:
  PASSWORD: changeme
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: NAME-pvc
  namespace: iot
spec:
  accessModes: ["ReadWriteOnce"]
  resources:
    requests:
      storage: 1Gi
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: NAME
  namespace: iot
spec:
  replicas: 1
  selector:
    matchLabels:
      app: NAME
  template:
    metadata:
      labels:
        app: NAME
    spec:
      containers:
        - name: NAME
          image: IMAGE
          env:
            - name: TZ
              value: Asia/Shanghai        # 避免容器 UTC 差 8 小时
          ports:
            - containerPort: PORT
          volumeMounts:
            - name: data
              mountPath: MOUNT
          readinessProbe:
            tcpSocket:
              port: PORT
            initialDelaySeconds: 10
            periodSeconds: 10
          livenessProbe:
            tcpSocket:
              port: PORT
            initialDelaySeconds: 30
            periodSeconds: 15
      volumes:
        - name: data
          persistentVolumeClaim:
            claimName: NAME-pvc
---
apiVersion: v1
kind: Service
metadata:
  name: NAME
  namespace: iot
spec:
  type: ClusterIP
  selector:
    app: NAME
  ports:
    - port: PORT
      targetPort: PORT
```

> 集群内其他服务用 **Service 名**（如 `mysql`、`redis`）当主机名即可互相访问（K8s DNS）。
