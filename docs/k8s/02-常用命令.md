# 02 · 常用命令速查

> 约定：本项目命名空间是 `iot`，命令多带 `-n iot`。可 `kubectl config set-context --current --namespace=iot` 设默认省去。

## 集群 / 节点（kind）

```bash
kind get clusters                              # 列集群
kind create cluster --name iot                 # 建集群
kind delete cluster --name iot                 # 删集群
kind load docker-image <镜像> --name iot        # 把宿主机镜像拷进节点（关键！）
kubectl get nodes -o wide                      # 节点状态
kubectl config current-context                 # 当前 context
```

## 部署 / 更新 / 删除

```bash
kubectl apply -f deploy/k8s/                    # 部署整个目录（按文件名字母序）
kubectl apply -f deploy/k8s/10-mysql.yaml       # 部署单个文件
kubectl delete -f deploy/k8s/10-mysql.yaml       # 按清单删除
kubectl rollout restart deploy/redis -n iot     # 重建 Pod 让新配置生效
kubectl rollout status deploy/redis -n iot      # 看滚动更新进度
kubectl rollout undo deploy/redis -n iot        # 回滚上一版
```

## 查看 / 诊断

```bash
kubectl get pods -n iot                         # Pod 列表
kubectl get pods -n iot -w                      # 持续监视（Ctrl+C 退出）
kubectl get all -n iot                          # 该命名空间所有资源
kubectl describe pod -n iot -l app=mysql        # 事件（拉镜像/探针/挂载失败都在这）
kubectl logs -n iot -l app=mysql                # 容器日志
kubectl logs -n iot <pod> --previous            # 上次崩溃的日志
kubectl get pvc,secret -n iot                   # 看 PVC / Secret
```

## 进容器排查

> `--` 之前是 kubectl 开关（如 `-it`），之后是容器内命令；命令带 `-` 参数时**必须**有 `--`。原理见 `06`。
>
> **`-i` / `-t` 怎么选**：`-i`=接通 stdin（喂输入）、`-t`=分配伪终端（人肉交互）。
> · 只看输出 → 不加；· 管道灌文件(`< f.sql`) → **只 `-i`（加 -t 会报 TTY 错）**；· 进 shell 自己敲 → `-it`。

```bash
kubectl exec -it -n iot deploy/redis -- sh      # 进 redis（alpine 只有 sh）
kubectl exec -it -n iot deploy/mysql -- bash    # 进 mysql（debian 有 bash）

# 进去后（普通 Linux）：
id                                              # 我这个 shell 是谁（≠主进程用户！）
ps                                              # 进程 + USER 列（alpine 有 ps）
cat /proc/1/status | grep -i uid                # 1 号进程真实 uid（精简镜像没 ps 时用）
getent passwd <uid>                             # uid → 用户名
ls -l /data ; stat <文件>                        # 权限 / 属主
```

## 进 kind 节点（很少用，兜底）

```bash
docker exec -it iot-control-plane bash          # 进节点
crictl ps                                       # 运行中的容器
crictl pods                                     # 所有 Pod
crictl images                                   # 节点内镜像（看 kind load 进来的）
crictl logs <容器ID>
exit
```

## 快速验证服务

```bash
# MySQL
kubectl exec -it -n iot deploy/mysql -- mysql -uroot -pexample123 -e "show databases;"
# Redis（靠 REDISCLI_AUTH 环境变量自动认证）
kubectl exec -it -n iot deploy/redis -- redis-cli ping
```
